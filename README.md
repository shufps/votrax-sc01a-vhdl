# SC01-A Votrax Formant Speech Synthesizer

FPGA implementation of the Votrax SC01-A phoneme speech synthesizer,
based on the MAME simulation by Olivier Galibert (BSD-3-Clause).

**note**: Assumptions about internal parameters (clock frequency,
capacitor values) are inherited from MAME and may not reflect the real chip exactly. Sounds "good enough" though.

## Repository Structure

```
rtl/
  *.vhd                  — SC01-A core (synthesizable, board-independent)
  astrocade/             — Bally Astrocade board-level RTL
sim/
  iir_filter_sim.vhd     — simulation-only IIR variant (coefficients as ports)
  astrocade/             — Bally Astrocade simulation (testbench, scripts, vectors)
mame-fp/                 — C++ reference simulation and vector generator
tools/                   — coefficient ROM generator and utility scripts
```

## Architecture

The SC01-A produces speech by chaining six IIR biquad filters driven
by a glottal waveform (voice path) and a noise source (noise path).
Filter coefficients are pre-computed as static ROM tables in s(2.15)
fixed-point — no floating-point or runtime division anywhere.

Signal path:
```
Voice:  glottal → scale(va) → F1 → F2V ──────────────────┐
Noise:  ±0.5    → scale(fa) → FN → scale(fc) → (F2N=0) ──┤
                                                         ↓
                mix → F3 → +noise_inj → F4 → scale(closure) → FX → out
```

## Filter Coefficients

IIR filter coefficients are precomputed using a prewarped bilinear transform.
Due to the fixed clock ratio (`sclock = mainclock/18`, `cclock = mainclock/36`),
the normalized peak frequencies `fpeak/Nyquist` are clock-independent — so the
coefficients are constant regardless of the actual oscillator frequency and can
be stored as fixed ROMs.

Each filter uses independently optimized fixed-point scaling: feedforward (A)
and feedback (B) coefficients are scaled separately to maximize precision within
18-bit signed range. The scale constants are emitted as a VHDL package
(`sc01a_coeff_scales_pkg.vhd`) and used as generics in `iir_filter_slow`.

Coefficients are verified stable (all poles strictly inside the unit circle)
across all filter parameter combinations.

Based on the MAME SC-01A implementation by Olivier Galibert (BSD-3-Clause),
with the following adaptations:
- Prewarping applied consistently to all filters including the noise shaper (FN)
- ROM generator verified against MAME reference output

### F2N (noise injection filter)

A noise injection path at the F2 output is present in the original hardware.
The F2N filter implementation (`build_injection`) required non-standard sign
handling to guarantee stability across all Q/F parameter combinations, as the
denominator sign varies with the operating point. F2N is disabled by default
pending further verification against real hardware recordings.

## Resource Usage

sc01a.vhd was synthesized with Quartus Lite 17.1 (no pin assignments; only a 50 MHz test clock) to evaluate resource usage.

<img width="493" height="435" alt="image" src="https://github.com/user-attachments/assets/28d0d38c-dbe6-4a65-b90d-6d8de8448ad6" />

The default 50 MHz target was met (Fmax ~76 MHz).

Note: The internal DDS (in `votrax_sound.vhd`) can be adjusted via the 8-bit DAC input, which changes the spacing of the generated sclock pulses. If the pulses get too close together, the filter chain (running on the normal system clock) may no longer have enough time to complete its processing between sclock events.

The clock can be configured via a generic. Clocks as low as ~20 MHz should still work across the usual clock-sweeping range.

| File | Description |
|------|-------------|
| `votrax_sound.vhd` | Astrocade Votrax wrapper: DDS clock generation + SC01-A core + 48 kHz resampler |
| `bally_rc_filter.vhd` | On-board RC lowpass — 4× cascaded (R=110 kΩ, C=560 pF), fc ≈ 2584 Hz, fs=48 kHz |

## Coefficient ROMs

All auto-generated by `tools/gen_votrax_roms.cpp`. Synchronous read, 1-cycle latency, infer as BRAM.

| File | Filter | Entries | Address |
|------|--------|---------|---------|
| `rtl/f1_rom.vhd` | F1 — first formant (voice) | 128 | `addr[6:3]` = freq step (0..15) |
| `rtl/f2v_rom.vhd` | F2V — second formant + Q (voice) | 4096 | `addr[11:3]` = {F2Q[3:0], F2[4:0]} |
| `rtl/f2n_rom.vhd` | F2N — noise injection filter | 4096 | same as F2V |
| `rtl/f3_rom.vhd` | F3 — third formant | 128 | `addr[6:3]` = freq step (0..15) |
| `rtl/f4_rom.vhd` | F4 — fourth formant (fixed) | 8 | — |
| `rtl/fn_rom.vhd` | FN — nasal formant / noise path (fixed) | 8 | — |
| `rtl/fx_rom.vhd` | FX — output / treble boost (fixed) | 8 | — |

Coefficient layout for all ROMs: `addr[2:0]` = coeff index:
`0=b0  1=a0  2=a1  3=a2  4=a3  5=b1  6=b2  7=b3`

Indices 1..N_X use `FP_FRAC_A` scaling (feedforward); indices 5..4+N_Y-1 use `FP_FRAC_B` scaling (feedback).
The scale constants for each filter are in `rtl/sc01a_coeff_scales_pkg.vhd`.

## Simulation

### Bally Astrocade (`sim/astrocade/`)

| File | Description |
|------|-------------|
| `bally.vhd` | Simulation wrapper: Bally Astrocade signal decoding + VotraxSound + RC filter |
| `bally_tb.vhd` | GHDL testbench — replays phoneme event vectors, writes `audio_out.raw` |
| `votrax_tb_vectors.vhd` | Input event vectors (auto-generated via `mame-fp/convert_to_vectors.py`, do not edit) |
| `bally_simulate.sh` | One-shot GHDL compile + simulate script |

### Running the simulation

```bash
cd sim/astrocade
./bally_simulate.sh
```

Produces `audio_out.raw` (signed 16-bit samples, one per line) and `wave.ghw`.

Convert to WAV for listening:
```bash
python3 tools/raw_to_wave.py sim/astrocade/audio_out.raw audio_out.wav
```

### `sim/iir_filter_sim.vhd`

Simulation-only IIR variant — coefficients passed directly as ports (no BRAM).
Useful for isolated filter testing without the full pipeline.

## mame-fp/

The `mame-fp/` directory contains the C++ reference simulation and tooling for generating GHDL testbench input vectors from real MAME captures.

| File | Description |
|------|-------------|
| `votrax.cpp` | Modified version of the MAME votrax.cpp (BSD-3-Clause, Copyright Olivier Galibert) — adapted to use s(2.15) fixed-point arithmetic and ROM-based coefficient lookup instead of floating-point; served as the initial feasibility test before the VHDL implementation |
| `votrax.h` | MAME votrax header (modified accordingly) |
| `votrax_rom_tables.h` | ROM parameter tables extracted from MAME |
| `votrax_dump.h` | Include in `votrax.cpp` (or MAME) to enable event logging — writes `votrax_input.txt` (phoneme/clock events) and `votrax_output.txt` (expected audio samples) |
| `convert_to_vectors.py` | Converts `votrax_input.txt` to `votrax_tb_vectors.vhd` for GHDL replay |

The C++ reference simulation was the first step: validating that the fixed-point math and ROM coefficient approach produce correct output before committing to GHDL. Faster iteration than the VHDL testbench for algorithm-level debugging.

### Workflow: capture from MAME → replay in GHDL

1. Copy the files from the `mame-fp` folder into MAME to `/src/devices/sound/` and compile
2. Run the C++ simulation — produces `votrax_input.txt` and `votrax_output.txt`
3. Convert to VHDL testbench vectors:
   ```bash
   python3 mame-fp/convert_to_vectors.py mame-fp/votrax_input.txt -o sim/astrocade/votrax_tb_vectors.vhd
   # optional: compress long silent gaps
   python3 mame-fp/convert_to_vectors.py mame-fp/votrax_input.txt --max-gap 500 -o sim/astrocade/votrax_tb_vectors.vhd
   ```
4. Run the GHDL simulation:
   ```bash
   cd sim/astrocade && ./bally_simulate.sh
   python3 tools/raw_to_wave.py sim/astrocade/audio_out.raw audio_out.wav
   ```

## tools/

| File | Description |
|------|-------------|
| `gen_votrax_roms.cpp` | Generates all `rtl/f*_rom.vhd` files and `rtl/sc01a_coeff_scales_pkg.vhd` — computes bilinear-transform IIR coefficients with per-filter optimal A/B scaling; also emits a C header (`/tmp/votrax_rom_tables.h`) for the MAME fixed-point reference |
| `raw_to_wave.py` | Converts `audio_out.raw` (text, one int16 per line) to a 48 kHz WAV file |
| `rom2vhdl.py` | Utility to convert raw ROM data to VHDL ROM entities |

### Regenerating the coefficient ROMs

```bash
cd tools
g++ -O2 -o gen_votrax_roms gen_votrax_roms.cpp -lm
./gen_votrax_roms
# writes all f*_rom.vhd, sc01a_coeff_scales_pkg.vhd and votrax_rom_tables.h to /tmp/
cp /tmp/f*_rom.vhd /tmp/sc01a_coeff_scales_pkg.vhd ../rtl/
```
